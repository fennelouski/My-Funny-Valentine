# Screenshots Directory

## Required Screenshots

You need screenshots for each device size. At minimum, provide 3 screenshots per device size.

## Device Sizes

### iPhone 6.7" Display
- **Size**: 1290x2796 pixels
- **Directory**: `iphone-67/`
- **Devices**: iPhone 15 Pro Max, iPhone 14 Pro Max, etc.

### iPhone 6.5" Display
- **Size**: 1284x2778 pixels
- **Directory**: `iphone-65/`
- **Devices**: iPhone 11 Pro Max, iPhone XS Max, etc.

### iPhone 5.5" Display
- **Size**: 1242x2208 pixels
- **Directory**: `iphone-55/`
- **Devices**: iPhone 8 Plus, iPhone 7 Plus, etc.

### iPad Pro 12.9" Display
- **Size**: 2048x2732 pixels
- **Directory**: `ipad-pro-129/`
- **Devices**: iPad Pro 12.9" (all generations)

## Screenshot Guidelines

### What to Show

1. **Card Creation Flow**
   - Show template selection
   - Show face detection in action
   - Show customization options

2. **AI Features**
   - Show AI saying generation
   - Show generated sayings list
   - Show premium features (if applicable)

3. **Final Results**
   - Show completed cards
   - Show sharing options
   - Show multiple card examples

### Best Practices

- Use actual app screenshots (not mockups)
- Show real content (not placeholders)
- Highlight key features
- Demonstrate user value
- Professional, polished appearance
- Consistent style across all screenshots
- No device frames (Apple adds them)
- No status bar text (can be removed)
- Clean, uncluttered interface

### Screenshot Order

1. **First Screenshot**: Main feature or hero screen
2. **Second Screenshot**: Key functionality
3. **Third Screenshot**: Additional features or results
4. **Additional**: More features or use cases

## Capturing Screenshots

### From Simulator
- Use Xcode's screenshot feature
- Or use Cmd+S in Simulator
- Ensure correct device size selected

### From Device
- Use device screenshot (Power + Volume Up)
- Transfer to Mac
- Resize if needed to exact dimensions

### Editing
- Can remove status bar if desired
- Can add text overlays (sparingly)
- Can highlight features
- Keep it authentic to the app

## File Naming

Use descriptive names:
- `screenshot-1-main-feature.png`
- `screenshot-2-ai-generation.png`
- `screenshot-3-card-results.png`
